OMNI PLAYLIST SYNC
SPEC NO. 001 — REV A · JULY 2026

The interface,
in careful hands.

A complete visual system for a self-hosted tool that touches people's music libraries and their credentials. The register is calm confidence — warm charcoal and paper neutrals, one instrument-panel accent, status always spelled out in words as well as color. Spotify, Apple Music, YouTube Music and Jellyfin appear only as small identity dots: guests in the house, never the wallpaper.

react + tailwind v4 · tokens → src/index.css @theme self-hosted · zero runtime CDNs light + dark, both first-class tweaks ▸ flip theme & accent — the whole document re-themes live
№ 01

Direction & the accent

four principles · three candidates · one recommendation
Calm over clever
One accent-colored action per view, ranked by commitment. Dry-run is quiet; execute is the loud one — and even it is just a solid button behind a confirm.
Status is spelled out
Every state carries glyph + label + color, never color alone. Feed kinds keep fixed glyphs (+ − ~ × !) so a colorblind user reads the same story.
Services are guests
Brand colors live only in 8–10px dots and soft badges. The app's own accent shares a hue with none of them, so this never reads as anyone's client.
Unhurried motion
120–320ms, ease-out, opacity + a 4px rise. The only loops are a soft pulse and a three-bar meter while a pass runs. Reduced motion collapses all of it to crossfades.
A · Signal Teal
RECOMMENDED DEFAULT
Spectrum-analyzer teal against warm charcoal — the needle color on a vintage meter face. Maximum hue distance from all four service brands and all four semantic colors. Reads "instrument", not "brand".

№ 10

Implementation — drop-in map

tokens → omni-theme.css · components → existing library · no new dependencies
WIRING
src/index.cssPaste the @theme block from omni-theme.css. Light values are the defaults; dark values override inside .dark via a @custom-variant. The toggle sets .dark on <html>, persists to localStorage, and defaults to prefers-color-scheme.
fonts@fontsource/schibsted-grotesk (400–800), @fontsource/instrument-serif (400 + italic), @fontsource/ibm-plex-mono (400–600) — bundled by Vite, zero runtime CDN. System stacks are declared as fallbacks in the tokens.
iconsNone required — status glyphs (+ − ~ × ! ✓ → ⇄) are set in Plex Mono. If pictographic icons arrive later, inline-SVG them at build time.
Buttonvariants primary · secondary · ghost · danger-ghost; sizes sm 32 / md 40 / lg 48; loading swaps label for spinner + verb.
StatusPillmap connected→success · expired→warning · unconfigured→neutral · error→danger, plus a checking spinner state.
TextFieldadds secret (masked + reveal), validating, and success/error adornment slots; helper text is a fixed-position slot so layout never jumps.
newfour small primitives to build once: FeedRow · CountChip · ServiceBadge · Segmented (PairCode is a styled span). Everything else is a re-skin of the existing library.
RESPONSIVE + ACCESSIBILITY CONTRACT
✓Mobile-first; drawer below lg; grids collapse to one column; no horizontal overflow to 320px — long tokens, feed text and playlist names wrap or scroll inside their container.
✓Tap targets ≥44px below md; inputs ≥16px there so iOS never zooms.
✓AA contrast in both themes; status = glyph + word + color everywhere; focus ring (2px gap + 2px accent) on every interactive element, never suppressed.
✓Feed is aria-live="polite"; counters announce totals, not every tick. Modals trap focus, Esc closes, destructive confirms never auto-focus the primary.
✓prefers-reduced-motion: entrances become crossfades; the meter and pulse hold still; nothing conveys meaning by motion alone.
One system, four services, no drama — the needle just moves.
